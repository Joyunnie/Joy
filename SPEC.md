# 고양이 생식 레시피 계산기 - 웹앱 개발 명세서

## 프로젝트 개요

기존 Excel 파일(고양이밥.xlsm)의 모든 기능을 웹앱으로 1:1 재구현한다.
첨부된 `food_data.json`과 `app_config.json`에 모든 원본 데이터와 설정이 있다.

**기술 스택**: React (Vite) + Tailwind CSS, 단일 페이지 앱, 서버 없음, 로컬에서 `npm run dev`로 실행.

---

## 1. 데이터 구조

### 1-1. 식품 영양소 데이터 (`food_data.json`)

- 230개 식품, 각각 57개 영양소 (100g 기준 수치)
- `foods[]` 배열: 각 항목에 `row`, `id`(드롭다운 인덱스), `name`, `nutrients{}` 포함
- `categories{}` 객체: 20개 드롭다운 카테고리별 식품 목록
  - 키: 식품A, 식품AA, 식품B, 식품C, 식품D, 식품F, 식품FF, 식품G, 식품H, 식품HH, 식품I, 식품J, 식품JJ, 식품K, 식품L, 식품M, 식품MM, 식품P, 식품Q, 식품R
  - 각 카테고리의 `items[]`에 `{index, name}` — index가 0이면 빈 선택, index ≥ 2가 실제 식품

### 1-2. NRC 기준값 / 앱 설정 (`app_config.json`)

- `nrc{}`: 영양소별 최소 필요량 (자묘/성묘 구분) + 상한선
  - 기본영양 15개, 무기질 9개, 아미노산 14개, 지방산 5개
- `calorie_formulas{}`: 6단계 칼로리 공식
- `dropdown_mapping[]`: 60개 드롭다운의 셀→카테고리 매핑
- `presets{}`: 피어슨/미쉘 프리셋 레시피

## 2. 입력부 UI (레이아웃 좌측)

### 2-1. 기본 정보 입력

| 항목 | 입력방식 | 기본값 |
|------|---------|--------|
| 고양이 현재 체중 (kg) | 숫자 input | 4 |
| 칼로리 타입 | 라디오버튼 6개 (아래 참조) | 2 |
| 성묘 때 예상체중 (kg) | 숫자 input (타입 5,6에서만 사용) | 4 |
| 레시피 일 수 | 숫자 input | 60 |
| 하루 필요칼로리 | 자동계산 표시 (편집 불가) | — |

**라디오버튼 6개 (B8 셀)**:
- 값을 라벨로 보여줘야 함. 원본 엑셀에서는 옵션 라벨이 보이지 않으므로, 다음과 같이 표시:
  - 1: 활동적 성묘 (110×W^0.4)
  - 2: 일반 성묘 (110×W^0.4×0.9) ← 기본값
  - 3: 비만 성묘 (100×W^0.67)
  - 4: 고활동 성묘 (130×W^0.4)
  - 5: 자묘 저활동 (성장공식×0.9)
  - 6: 자묘 일반 (성장공식)

**칼로리 공식 (JavaScript)**:
```javascript
function calcCalories(type, weight, expectedWeight) {
  switch(type) {
    case 1: return 110 * Math.pow(weight, 0.4);
    case 2: return 110 * Math.pow(weight, 0.4) * 0.9;
    case 3: return 100 * Math.pow(weight, 0.67);
    case 4: return 130 * Math.pow(weight, 0.4);
    case 5: return 100 * Math.pow(weight, 0.67) * 6.732 * (Math.exp(-0.189 * weight / expectedWeight) - 0.66) * 0.9;
    case 6: return 100 * Math.pow(weight, 0.67) * 6.732 * (Math.exp(-0.189 * weight / expectedWeight) - 0.66);
    default: return 0;
  }
}
```

## 3. 계산 엔진

### 3-1. 핵심 계산 흐름

```
1. 각 슬롯에서 드롭다운으로 식품 선택 → index로 food_data에서 해당 식품의 영양소 데이터 조회
2. 각 슬롯: 영양소값 = (식품의 100g당 영양소값 / 100) × 사용량(g)
3. 57개 영양소 각각에 대해 전체 슬롯 합산 → 총량
4. 하루 섭취량 = 총량 / 레시피 일수
5. 과부족% = 하루 섭취량 / (NRC기준값/1000 × 하루필요칼로리)
```

### 3-2. 드롭다운 index → 식품명 조회

드롭다운의 값(1,2,3...)은 해당 카테고리 items 배열에서의 1-based 순서다.
- index 0 = 선택 안 함
- index 1 = 카테고리의 첫 번째 항목 (보통 빈 항목)
- index 2+ = 실제 식품

```javascript
function getFoodByDropdown(categoryKey, dropdownIndex) {
  if (dropdownIndex <= 0) return null;
  const category = categories[categoryKey];
  // items 배열에서 해당 index의 식품명을 찾음
  const item = category.items.find(i => i.index === dropdownIndex);
  if (!item) return null;
  // food_data에서 해당 식품명으로 영양소 데이터 조회
  return foods.find(f => f.name === item.name);
}
```

### 3-3. 영양소 합산

```javascript
function calcTotalNutrients(allSlots) {
  const totals = {};  // 57개 영양소 각각 0으로 초기화
  
  for (const slot of allSlots) {
    const food = getFoodByDropdown(slot.category, slot.dropdownValue);
    if (!food || !slot.amount || slot.amount <= 0) continue;
    
    for (const [nutrient, value] of Object.entries(food.nutrients)) {
      if (typeof value === 'number') {
        totals[nutrient] = (totals[nutrient] || 0) + (value / 100) * slot.amount;
      }
    }
  }
  return totals;
}
```

### 3-4. 과부족% 계산

```javascript
function calcSufficiency(dailyAmount, nrcValue, dailyCalories) {
  // nrcValue: NRC 기준값 (per 1000Kcal)
  // dailyCalories: 하루 필요칼로리
  const requirement = (nrcValue / 1000) * dailyCalories;
  if (requirement <= 0) return null;
  return dailyAmount / requirement;  // 1.0 = 100% 충족
}
```

**자묘/성묘 분기**: B8 값이 5 또는 6이면 자묘(kitten) NRC 기준 사용, 그 외 성묘 기준 사용.

### 3-5. 경고 메시지 조건

| 조건 | 메시지 |
|------|--------|
| 칼로리 < 필요칼로리 | ▶칼로리 낮음 |
| 수분량 < 필요량×0.9 | ▶수분 추가섭취 필요 |
| 칼슘 DM% > 4% | ▶칼슘 높음 |
| 인:칼슘 비율 < 0.5 | ▶칼슘 비율 낮음 |
| 인:칼슘 비율 > 1.5 | ▶칼슘 비율 높음 |
| 각 영양소 > 상한선 | ▶{영양소명} 높음 |
| 특정영양소 조절값 초과 | ▶조절 {영양소명} 초과 |
| 셀레늄 > 228% | ▶셀레늄 높음 (빨간색) |

### 3-6. 비율 계산

- **인:칼슘비 = 칼슘(mg) / 인(mg)**
- **뼈:살 비율 = (생뼈사용량 + RMB사용량×0.6) / (생뼈 + RMB + 고기류 전체 + 내장류 전체)**
- **내장:육류 비율 = 내장류 합계 / (생뼈 + 고기류 + 내장류 합계)**
- **퓨레:육류 비율 = (채소류 + 기타야채퓨레 합계) / (생뼈 + RMB + 고기류 + 내장류 + 채소류 + 기타야채퓨레)**

### 3-7. 수분량 경고 계산

```javascript
const 수분량 = dailyNutrients['수분(g)'];
const 탄수화물 = dailyNutrients['탄수화물(g)'];

const 단백질 = dailyNutrients['단백질(g)'];

const totalWater = 탄수화물 * 0.6 + 지방 * 1.08 + 단백질 * 0.36 + 수분량;
const requiredWater = dailyCalories * 0.9;

if (totalWater < requiredWater) {
  // "▶수분 추가섭취 필요"
  // "하루필요 수분량 {requiredWater}g중"
  // "{requiredWater - totalWater}g의 추가 음수 필요"
}
```

## 7. 검증 기준

완성 후 피어슨 프리셋을 로드했을 때, 다음 값과 일치해야 함 (체중 3.8kg, 타입 2, 레시피 11일 기준):

- 하루 필요칼로리: 약 168.87 Kcal
- 이 값은 체중 3.8, 타입 2 기준: 110 × 3.8^0.4 × 0.9 ≈ 168.87

현재 엑셀 파일의 실제 결과값 (체중 3.8kg, 타입 2, 60일분 기준):

## 9. 주의사항

1. **드롭다운 index는 1-based**. 엑셀 Form Control 콤보박스는 선택 시 1부터 시작. index=1이면 카테고리의 첫 항목(보통 빈칸), index=2부터 실제 식품.
2. **영양소 데이터는 100g 기준**. 사용량이 g 단위가 아닌 것들(캡슐, tsp, 겔 등)은 자료시트에서 이미 1단위 기준으로 환산되어 있으므로, 사용량 × (영양소/100)이 아니라 **사용량 × 영양소값** 그대로가 맞는 슬롯도 있다. → `food_data.json`의 `함량(g)` 필드를 확인해서 100g 기준인지 1단위 기준인지 구분할 것. "100g"이면 /100, 그 외(예: "1캡슐", "1겔" 등)이면 그대로 곱하기.
3. **과부족%에서 자묘/성묘 분기**: B8(칼로리타입)이 5 또는 6이면 NRC 자묘 기준, 그 외 성묘 기준.
4. **비율 계산에서 0 나누기 방지**: IFERROR 처리 필수.
5. **모든 수치는 소수점 1자리**까지 표시 (과부족%는 정수%).
